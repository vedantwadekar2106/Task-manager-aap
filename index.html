🔬 Attributes vs Properties Demonstration

The value property gives the current live value of the input. getAttribute("value") returns the original HTML attribute value (the default). Type in the input below and click the button to see the difference in the console.

Live .value: Original HTML Value

🔼 Event Bubbling

Event travels from the target element upward through ancestors.

Order: Child → Parent → Grandparent

This is the default behavior (capture: false).

🔽 Event Capturing

Event travels from the outermost ancestor down to the target.

Order: Grandparent → Parent → Child

Enabled with addEventListener(type, handler, true).

🖥️ Browser Rendering Pipeline

Understanding how the browser converts HTML, CSS, and JavaScript into pixels on the screen.

📄

HTML

Raw HTML markup received from the server or generated dynamically.

⬇️
🔍

Parsing

The browser's HTML parser reads the raw bytes and converts them into meaningful tokens.

⬇️
🧩

Tokenization

Characters are grouped into tokens: start tags, end tags, attributes, text content.

⬇️
🌳

DOM Tree

Tokens are assembled into a hierarchical tree structure — the Document Object Model.

➕
🎨

CSS

Stylesheets are parsed into the CSS Object Model (CSSOM Tree).

⬇️
🎭

Render Tree

DOM Tree + CSSOM Tree combine to form the Render Tree — only visible elements.

⬇️
📐

Layout (Reflow)

Calculates exact position and size of each element on the viewport.

⬇️
🖌️

Paint

Fills pixels with colors, text, images, borders, shadows — rasterization.

⬇️
🖼️

Composite

Layers are combined and drawn to the screen in the correct order.

📌 Summary Flow: HTML → Parsing → Tokenization → DOM Tree  +  CSS → CSSOM Tree  →  Render Tree → Layout → Paint → Composite → Screen
